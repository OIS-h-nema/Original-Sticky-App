# 付箋アプリをローカルPCに実装するまでの手順書

本手順書は、付箋アプリをローカルPC（Windows）に実装・動作させるまでの作業フローをまとめたものです。要件・設計内容は `設計資料` フォルダの文書に準拠します。

## 1. 事前準備

### 1.1 必要なソフトウェア
- Windows 10/11
- .NET 8 SDK
- Visual Studio 2022（「.NET デスクトップ開発」ワークロード）
- Git

### 1.2 設計資料の確認
以下の資料を参照し、実装方針・機能要件・画面仕様を確認します。
- `設計資料/付箋アプリ_要件定義書.docx`
- `設計資料/付箋アプリ_基本設計書_v1.1.docx`
- `設計資料/付箋アプリ_詳細設計書_v1.0.docx`

## 2. リポジトリの準備

1. Gitで本リポジトリを取得します。
   ```
   git clone <リポジトリURL>
   cd Original-Sticky-App
   ```
2. ローカル作業用のブランチを作成します（任意）。
   ```
   git checkout -b feature/local-implementation
   ```

## 3. プロジェクト作成

1. Visual Studio 2022 を起動し、**WPF アプリケーション (.NET)** を新規作成します。
2. プロジェクト名は「StickyNoteApp」とし、フレームワークは **.NET 8** を選択します。
3. 作成したプロジェクト一式を本リポジトリのルートに配置します。
   - 例: `StickyNoteApp/` ディレクトリを作成

## 4. 基本構成の実装

基本設計書にある「シンプルMVVM構成」に従い、以下を実装します。

### 4.1 ファイル構成（例）
```
StickyNoteApp/
├── App.xaml
├── App.xaml.cs
├── MainWindow.xaml
├── MainWindow.xaml.cs
├── SettingsWindow.xaml
├── SettingsWindow.xaml.cs
├── ViewModels/
│   ├── MainViewModel.cs
│   └── SettingsViewModel.cs
├── Models/
│   └── SettingsModel.cs
├── Services/
│   └── SettingsService.cs
└── Constants.cs
```

### 4.2 実装方針
- MVVMは最小限に留め、外部フレームワークは使用しない。
- 設定値はJSONで保存し、アプリ起動時に読み込む。
- Markdownのレンダリングは編集終了時にのみ行う。

## 5. 機能実装

### 5.1 付箋ウィンドウ
- 透明背景のウィンドウを作成
- ドラッグで移動可能
- 固定モードON/OFFで移動制御

### 5.2 テキスト入力と保存
- クリックで編集開始
- フォーカス離脱で自動保存
- Markdownを簡易レンダリング（太字・斜体・リストのみ）

### 5.3 設定画面
- トレイメニューから設定画面を開く
- 以下を設定可能にする
  - 幅 / 高さ
  - フォント
  - フォントサイズ
  - 文字色
  - 位置固定ON/OFF

### 5.4 常駐機能
- システムトレイに常駐
- トレイメニューに「設定」「終了」を追加

## 6. データ保存

- 保存先: `%AppData%\StickyNoteApp\settings.json`
- 保存内容: 位置・サイズ・テキスト・フォント設定など
- 詳細なJSON構造は要件定義書を参照

## 7. ビルドと動作確認

1. Visual Studio で **ビルド** を実行
2. アプリを起動し以下を確認
   - 付箋表示と編集
   - Markdown描画
   - 設定値保存と再起動時の復元
   - トレイ機能

## 8. 実装完了後の整理

- 実装ソースコードを `StickyNoteApp/` 以下に整理
- 不要ファイル（テンプレートのサンプル等）があれば削除
- Gitで変更内容をコミット

---

以上でローカルPC上で付箋アプリの実装と動作確認が可能になります。
